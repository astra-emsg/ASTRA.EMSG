Dialogues

Le traitement des données a généralement lieu par l'intermédiaire de boîtes de dialogues.

Celles-ci intègrent les boutons suivants:

Le système enregistre les données saisies. La boîte de dialogue demeure ouverte.

Le système interrompt le traitement. En l'absence de données saisies à enregistrer, la boîte de dialogue est refermée.

Le système enregistre les données saisies et ferme la boîte de dialogue.

Le système supprime (après demande de confirmation) le jeu de données.

 

Avertissement: Les jeux de données sont physiquement effacés, c'est-à-dire qu'il n'y a pas de «corbeille» à partir de laquelle il est possible de récupérer des jeux de données supprimés par erreur. Les jeux de données dépendants sont également perdus.

Validation

Lors de la saisie ou de la modification de données, la distinction est faite entre champs obligatoires et champs facultatifs (voir Figure 3).

Figure 3: Champ obligatoire (Date de relevé) et champ facultatif (Équipe effectuant le relevé)

Si vous tentez d'enregistrer les données sans avoir renseigné tous les champs obligatoires, la ligne est alors marquée et une indication d'erreur (point d'exclamation rouge) apparaît à droite des champs obligatoires concernés (voir Figure 4). Si vous déplacez la souris sur l'indication d'erreur, le message d'erreur correspondant apparaît.

Figure 4: Validation d'un champ obligatoire

De plus, en fonction du champ de données, il peut y avoir encore d'autres validations (par exemple la plage de valeurs d'un champ numérique) qu'il convient de traiter de manière similaire.

Champs de date

Les champs de date sont reconnaissables par un petit symbole de calendrier à droite du champ de saisie. Un clic sur le symbole de calendrier ouvre un calendrier permettant de sélectionner la date (voir

Figure 5). L'utilisateur peut alors soit remplir le champ de données au clavier, soit sélectionner la date au moyen de la souris.

EMSG-Mobile a en outre prérempli le champ de date avec la «Date de relevé».
